How often is the flight price data crawled?

The pricing engine crawls Google Flights automatically thrice a day at 8:00 AM, 2:00 PM, and 8:00 PM local time using Windows Task Scheduler (or via GitHub Actions in the cloud repository).

How do the price recommendations (BUY NOW vs WAIT) work?

Recommendations are calculated by comparing the latest crawl price with the running average for that specific route and lead-time:

  • BUY NOW (Green Trend): Current price is at least 3% below the historical average.
  • WAIT (Red Trend): Current price is at least 3% above the historical average.
  • BUY NOW (Stable Price): Current price is within 3% of the average, indicating stable booking conditions.
How do I sync my local price database with GitHub Pages?

If you run the scraper locally, you can easily sync your database and update the online dashboard by pushing the updated JSON to GitHub:

git add flights_db.json
git commit -m "Update flight prices database"
git push origin main

This automatically triggers a GitHub Pages deployment and updates your online tracker site.
