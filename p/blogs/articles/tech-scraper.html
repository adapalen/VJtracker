Tự động hóa theo dõi giá vé máy bay: Puppeteer & GitHub Actions

Theo dõi sự biến động của giá vé máy bay là một bài toán thú vị. Làm thế nào để tự động hóa quy trình này một cách ổn định, tối ưu chi phí hạ tầng và thu thập dữ liệu sạch? Bài viết dưới đây sẽ chia sẻ kiến trúc chi tiết.

1. Giới thiệu tổng quan kiến trúc

Hệ thống được thiết kế chạy hoàn toàn phi máy chủ (Serverless) để tiết kiệm chi phí vận hành. Toàn bộ logic quét dữ liệu được lập trình bằng Node.js và thư viện điều khiển trình duyệt không đầu (Headless browser) Puppeteer.

Mỗi ngày 3 lần, các bộ kích hoạt tự động (Workflow triggers) của GitHub Actions sẽ khởi tạo một máy ảo Linux tạm thời, tải mã nguồn, chạy chương trình quét dữ liệu giá vé máy bay thực tế từ Google Flights, và lưu trữ kết quả thẳng vào file cơ sở dữ liệu JSON tĩnh.

2. Vượt qua cơ chế chặn (Anti-bot mechanisms)

Các hệ thống OTA lớn như Google Flights tích hợp các thuật toán phát hiện bot tự động rất nghiêm ngặt (Cloudflare, Akamai). Để chương trình chạy mượt mà, chúng tôi áp dụng các giải pháp:

  • Sử dụng plugin puppeteer-extra-plugin-stealth để ẩn dấu vết webdriver.
  • Tùy biến User-Agent ngẫu nhiên và mô phỏng chính xác các tham số phần cứng màn hình/độ phân giải.
  • Sắp xếp thời gian trễ ngẫu nhiên giữa các thao tác nhấp chuột và nhập liệu văn bản để giả lập con người.
  • Ép buộc các tham số ngôn ngữ và tiền tệ trong URL truy cập để đảm bảo cấu trúc HTML trả về đồng bộ (hl=vi&gl=VN&curr=VND).

3. Triển khai CI/CD trên GitHub Actions

File cấu hình workflow yaml định cấu hình một môi trường Ubuntu chạy ổn định với Node.js 24. Sau khi tiến trình cào hoàn tất và ghi đè dữ liệu mới vào flights_db.json, Git sẽ tự động tạo commit mới và đẩy ngược dữ liệu lên kho chứa (Repository) tự động, giúp cập nhật biểu đồ phân tích lập tức.
